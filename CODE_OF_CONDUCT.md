# Code of Conduct

## Our standard

AgentMesh is a public-good protocol. The artifact is the protocol; the
participants are professionals collaborating on it.

We expect:

- **Direct communication** — say what you mean, ask for what you need
- **Technical rigour** — disagreement is fine; back claims with code, tests,
  or specifications
- **Good faith** — assume the other person is trying to make the protocol
  better, not undermine you
- **Respect for time** — read the docs before opening an issue, run the
  tests before claiming a bug, search closed issues before posting

We do not tolerate:

- Personal attacks, harassment, or discrimination of any kind
- Trolling or deliberately-bad-faith argumentation
- Doxxing or sharing private information
- Spam, off-topic promotion, or unsolicited DMs
- Token shilling, paid endorsements, or attempts to manipulate the project
  for financial gain at the expense of users

## Enforcement

Maintainers may, at their discretion:

- Edit, hide, or remove comments / issues / PRs that violate the standard
- Lock threads that have become unproductive
- Block users who repeatedly violate after warning

There is no formal "review committee" — the maintainers act in good faith
based on the public record. If you believe an enforcement action was wrong,
open a discussion thread; the conversation is the appeal.

## Scope

This applies to all project spaces: GitHub (issues, PRs, discussions, code
review), official Discord / Slack channels (when those exist), and any
real-world events organised under the project's name.

## Reporting

For Code of Conduct violations: open a private GitHub advisory or contact
the maintainer directly. We do not require a victim to be the reporter —
if you saw something off, you can flag it.

For security issues: see [`SECURITY.md`](SECURITY.md).

## Spirit, not letter

This document is a sketch of expectations, not a legal contract. The goal
is healthy collaboration on a public-good protocol. Rules exist to serve
that goal, not to be litigated.
